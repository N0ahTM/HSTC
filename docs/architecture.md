# HSTC Amplify Gen 2 Architecture

## Overview
The HSTC site is delivered as an AWS Amplify Gen 2-hosted, Vite-powered React application. Amplify Gen 2 provides managed authentication, API, storage, hosting, and analytics services through a TypeScript-first, code-based approach. The frontend consumes these capabilities through the modern Amplify JavaScript v6 libraries.

## Core Services
- **Amplify Hosting** – Builds and serves the static assets generated by Vite and enables PR previews.
- **Amazon Cognito (Auth)** – Provides secure sign-up/sign-in flows using the Amplify Authenticator UI. This gates member-only dashboards and contextual UI.
- **AWS AppSync (GraphQL API)** – Supplies the `submitProspect` mutation that the join/contact form invokes and exposes queries for content such as news or fleet data.
- **AWS Lambda** – Backing resolvers for AppSync mutations and scheduled Discord sync jobs.
- **Amazon DynamoDB** – Stores community updates, fleet rosters, and contact lead submissions.
- **Amazon S3** – Optional asset storage with CloudFront-backed delivery and image optimization presets.
- **Amazon Pinpoint (Analytics)** – Captures client-side events (CTA clicks, auth conversions) for growth tracking.

## Frontend Composition
- **Entry** – `src/main.tsx` bootstraps React and initializes the application. For Amplify Gen 2, configuration is consumed from `amplify_outputs.json` when backend resources are defined.
- **Layout** – `App.tsx` wires navigation, context providers, and top-level routes.
- **Sections** – Hero, Pillars, Fleet, Community, Events, Discord, Join, and Footer sections live in `src/sections/` with shared UI primitives in `src/components/`.
- **State & Data** – Custom hooks in `src/hooks/` can encapsulate Amplify Gen 2 interactions (auth status, Discord stats, contact submission) when backend is implemented.
- **Styling** – CSS Modules layered over a design token file in `src/styles/tokens.css`; global resets in `src/styles/global.css`.
- **Testing** – Vitest + Testing Library cover rendering and form submission logic.

## Environments & Configuration

### Amplify Gen 2 Configuration
Amplify Gen 2 uses a **code-first** approach with TypeScript. Backend resources are defined in the `amplify/` directory:

```
amplify/
├── auth/
│   └── resource.ts       # Cognito auth configuration
├── data/
│   └── resource.ts       # AppSync API and DynamoDB schema
├── storage/
│   └── resource.ts       # S3 bucket configuration
├── backend.ts            # Main backend definition
├── package.json
└── tsconfig.json
```

The build process generates `amplify_outputs.json` (Gen 2) which replaces the legacy `amplifyconfiguration.json` (Gen 1). This file is automatically consumed by the Amplify JavaScript v6 library when you call `Amplify.configure()`.

### Environment Variables
Additional environment variables (prefixed with `VITE_`) can be injected via Amplify build settings or `.env.local` for developers:

| Variable | Purpose |
| --- | --- |
| `VITE_DISCORD_WIDGET_URL` | Discord widget JSON endpoint |

**Note**: With Amplify Gen 2, authentication and API configurations are automatically provided through `amplify_outputs.json`, eliminating the need for manual environment variables for User Pool IDs, endpoints, etc.

## Deployment Flow
1. Developers commit to the `Development` branch.
2. Amplify Hosting automatically detects the `amplify/` backend directory and provisions resources using the Amplify Gen 2 backend-deploy pipeline.
3. Backend resources (Auth, Data, Storage, Functions) are deployed via CDK synthesis from TypeScript definitions.
4. The `amplify_outputs.json` file is generated and made available to the frontend build.
5. Amplify Hosting runs the frontend build (see `amplify.yml`) and uploads the Vite build output.
6. Preview deployments are auto-created for feature branches with isolated backend sandboxes.

**Key Differences from Gen 1**: 
- No Amplify CLI required - backend is defined entirely in TypeScript code
- Infrastructure as Code (IaC) using AWS CDK under the hood
- `amplify_outputs.json` replaces `amplifyconfiguration.json`
- Type-safe backend definitions with IDE autocomplete

## Security Considerations
- All secrets are supplied via Amplify environment variables and never committed to source control.
- Authenticated API flows require Amplify-managed tokens.
- Discord widget fetch is read-only; rate limiting is handled server-side by the Lambda job that mirrors data if needed.

## Scaling & Performance
- Vite produces optimized ES modules and code splitting for fast loading.
- Amplify Hosting deploys behind Amazon CloudFront for global caching.
- Images are served from the `public/images` directory with lazy loading and modern formats.
- Optional S3 image pipeline can further reduce load times with responsive image variants.
